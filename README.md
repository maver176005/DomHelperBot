## Стек

- Node.js
- `telegraf`
- локальное JSON-хранилище в `data/db.json`

## Запуск

1. Установите Node.js 18+.
2. Создайте Telegram-бота через BotFather и получите токен.
3. Установите зависимости:

```bash
npm install
```

4. Создайте `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

5. Укажите токен бота в `.env`:

```dotenv
BOT_TOKEN=your_telegram_bot_token
```

Также можно вместо `.env` экспортировать переменную окружения вручную:

```bash
export BOT_TOKEN=your_telegram_bot_token
```

6. Запустите проект:

```bash
npm start
```

## Как проверить MVP

1. Откройте бота и вызовите `/start`.
2. Пройдите регистрацию:
   имя, телефон, роль, дом, подъезд, этаж, квартира.
3. Для роли `client` создайте запрос через кнопку `🙋 Мне нужна помощь`.
4. Выберите тип запроса.
5. Для `🗑 Вынести мусор` укажите количество пакетов, комментарий, цену, оплату и отправьте фото до.
6. Для других типов укажите текст запроса, цену и выберите оплату кнопками.
7. Под другим Telegram-пользователем зарегистрируйтесь как `provider` в том же доме.
8. Исполнитель откроет `🔥 Запросы дома` и увидит только активные запросы своего дома.
9. После взятия запроса он увидит точную квартиру.
10. Для мусора исполнитель отправляет фото после, для других запросов просто отмечает выполнение.
11. Клиент подтверждает выполнение.

## Данные

- При первом запуске автоматически создается `data/db.json`.
- В файле уже есть 2 тестовых дома.
- Для MVP дома задаются локально и могут быть изменены вручную в `data/db.json`.

## Ограничения MVP

- Нет полноценной БД.
- Нет веб-админки.
- Нет платежного модуля.
- Регистрация и дома упрощены для локального запуска.

# STATUS.md

Текущее состояние реализации относительно `PROJECT.md` и `TASK.md`.

## Реализовано

- локальный запуск Telegram-бота
- автоматическая загрузка `BOT_TOKEN` из `.env`
- `/start`
- регистрация пользователя
- привязка пользователя к дому
- локальный справочник домов
- общий сценарий соседского запроса
- выбор типа запроса
- сценарий заказа `trash_removal`
- упрощенные сервисные запросы без фото
- обязательное фото до создания заказа
- house-scoped уведомления исполнителей
- защита от двойного взятия заказа
- скрытие квартиры до назначения исполнителя
- обязательное фото после выполнения
- подтверждение заказа клиентом
- карточка заказа
- отмена заказа клиентом до назначения исполнителя
- повтор завершенного заказа с новым фото до
- разделение заказов на активные и завершенные
- редактирование профиля
- быстрое переключение роли `client/provider`
- выбор способа оплаты кнопками
- будущая web-версия зафиксирована как следующий продуктовый этап
- заготовка под услуги и аренду
- общие доменные словари вынесены в `src/config/app-data.js`
- базовый интерфейсный словарь вынесен в `src/config/ui-copy.js`
- общая доменная логика заказов вынесена в `src/domain/order-helpers.js`
- зафиксирован go-to-market через объявление в подъезде и QR-код

## Бизнес-правила, которые уже соблюдаются

- действия изолированы по `houseId`
- исполнитель может брать только заказ своего дома
- один заказ не может быть взят двумя исполнителями
- фото до обязательно
- фото после обязательно
- квартира скрыта до взятия заказа
- после взятия заказа детали раскрываются исполнителю

## Осознанные MVP-упрощения

- локальное JSON-хранилище вместо БД
- дома задаются локально
- нет подтверждения реального проживания в доме
- нет house admin / super admin интерфейса
- нет платежной логики
- нет модуля услуг
- нет модуля аренды
- нет тестов

## Что просится следующим шагом

- реальные дома для теста в Обнинске
- web-версия с красивым UI
- landing / printable poster под запуск по подъездам
- вынесение текстов в отдельный модуль
- вынесение JSON-логики в отдельный storage-модуль
- базовый модуль `Listing` для “Я предлагаю”
- минимальная проверка данных пользователя
